17 事件类型之鼠标事件；Event事件对象；键盘事件

一、点击类事件（最常用）

事件名 触发时机 注意
click 鼠标左键（或键盘 Enter）在元素上完成一次“按下+释放” 触摸屏也会触发
dblclick 短时间内连续两次 click 实际中较少用，可模拟
mousedown 任意鼠标按键被按下（左/中/右键） 组合 event.button 判断键
mouseup 任意鼠标按键被释放 与 mousedown 配对
contextmenu 右键点击（触发上下文菜单前） 常用于自定义右键菜单

event.button 值：

  • 0 → 左键

  • 1 → 中键（滚轮）

  • 2 → 右键

二、移动类事件

事件名 触发时机 注意
mousemove 鼠标在元素内部移动时持续触发 即使鼠标按下也会触发
mouseenter 鼠标首次进入元素边界 不冒泡，不会在子元素间反复触发
mouseleave 鼠标离开元素边界 不冒泡
mouseover 鼠标进入元素（或其子元素）时 会冒泡，进入子元素也会再次触发
mouseout 鼠标离开元素（或进入子元素时） 会冒泡，行为类似 mouseover 的离开版

记忆技巧：
mouseenter/leave 更符合直觉（不冒泡，不会被内部子元素干扰），mouseover/out 更底层。

三、滚轮事件

事件名 触发时机 接口 注意
wheel 鼠标滚轮滚动时 WheelEvent 现代标准，取代旧事件
mousewheel （废弃）火狐用 DOMMouseScroll — 请一律使用 wheel

wheel 事件的 event.deltaY 属性表示垂直滚动方向和量（正值向下）。

四、拖拽相关事件（HTML5 Drag & Drop）

这些事件并非纯粹的“鼠标事件”，但依赖鼠标操作，常用于实现拖拽。

事件名 触发对象 说明
dragstart 被拖拽元素 开始拖拽（必须设置 draggable="true"）
drag 被拖拽元素 拖拽过程中持续触发
dragend 被拖拽元素 拖拽结束（无论成功与否）
dragenter 放置目标 拖拽元素进入目标区域
dragover 放置目标 拖拽元素在目标上移动（需阻止默认行为才能 drop）
dragleave 放置目标 拖拽元素离开目标区域
drop 放置目标 在目标上释放鼠标（需阻止 dragover 的默认行为）

五、其他辅助事件

事件名 说明
mouseleave 前还有 pointerleave 等指针事件 但属于 Pointer Events API
selectstart 鼠标开始选择文本时（并非鼠标事件本身，但关联

Event事件对象

事件发生以后，会产生一个事件对象，作为参数传给监听函数。

Event对象属性
event.target 点击谁，就打印谁

event对象方法
event.preventDefault()取消浏览器对事件的默认行为。
event.stopPropagation()阻止事件在DOM中继续传播，防止再触发定义在别的节点上的监听函数，但是不包括在当前节点上其他的事件监听函数（防止重叠的按钮连续触发事件，即事件嵌套）

键盘事件

主要有三个
keydown:按下键盘触发
keypress:按下有值的键时触发
keyup:松开键盘时触发事件

event对象

keyCode：唯一标识
回车按键的keyCode为13

18 事件类型之表单事件

input事件：一边输入，一边得到
select事件
Change事件
reset事件:重置表单
submit事件

1. submit – 表单提交

触发时机：用户点击提交按钮（<button type="submit">）或在输入框按 Enter 键。
常见用途：

  • 阻止默认页面跳转，改用 Ajax 发送数据

  • 做最终的表单验证，不通过则阻止提交

2. input – 输入内容实时变化

触发时机：<input>、<textarea> 或 <select> 的值每次改变时立刻触发（包括键盘输入、粘贴、剪切等）。
常见用途：

  • 实时搜索联想

  • 字符计数、输入预览

  • 即时校验提示

注意：和 change 的区别是，input 每输入一个字符就触发，而 change 要等失去焦点（对文本输入框）或选择改变（对选择框）才触发。

3. change – 值被最终确认

触发时机：

  • 对 <input type="text"> 等：用户修改内容并失去焦点时触发

  • 对 <select>、checkbox、radio：选中项发生改变时立即触发
    常见用途：

  • 下拉菜单联动

  • 文件选择后预览

  • 表单最终值校验（不需要实时干扰）

4. focus – 获得焦点

触发时机：元素成为可交互的焦点（鼠标点击、Tab 键切换、element.focus() 等）。
常见用途：

  • 显示输入提示 / 占位文字

  • 高亮边框，增强交互感

  • 打开自定义下拉菜单

特点：focus 事件不冒泡，如需冒泡可以监听 focusin。

5. blur – 失去焦点

触发时机：元素失去焦点（点击其他地方、Tab 离开、element.blur() 等）。
常见用途：

  • 校验表单字段（如邮箱格式）

  • 隐藏提示信息 / 收拢 UI

  • 提交自动保存

同样不冒泡，可用 focusout 代替。

总结对比表

事件 触发特点 典型场景
submit 表单整体提交 Ajax 提交、验证
input 值每次改变立即触发 实时搜索、字符计数
change 值改变后失去焦点或选项切换 下拉联动、文件选取
focus 获得焦点 显示提示、UI 高亮
blur 失去焦点 校验、保存、UI 收起

20 定时器setTimeout

setTimeout 是 JavaScript 中最基础的异步定时器，用于延迟执行一段代码。它属于浏览器（和 Node.js）的 Web API，与事件循环紧密配合。
#基本语法

  • 第一个参数：要执行的函数（或字符串代码，但不推荐）。

  • 第二个参数：延迟的时间，单位毫秒（1秒 = 1000）。

  • 后续参数：会作为回调函数的实参传入。

#返回值与清除定时器
setTimeout 会返回一个定时器 ID（浏览器中是正整数），用于取消该定时器。

21 定时器setInterval

setInterval 与 setTimeout 一样是异步定时器，但它会每隔指定时间重复执行回调，直到被清除。

返回值与清除定时器
setInterval 也返回一个定时器 ID，必须用 clearInterval 清除，否则会一直执行下去。

如果没有及时清除，且回调引用了外部变量，会导致内存泄漏。

22 防抖（debounce)

防抖（debounce） 是前端性能优化的核心手段之一，它的本质是将多次高频执行的操作合并为一次，只在“操作停止”一段时间后才真正执行。

23 节流（throttle）

节流（throttle）也是高频事件优化的核心手段，和防抖是一对“双胞胎”。防抖是“等你安静下来”，节流是“按固定节奏执行”。

1. 时间戳版（第一次立即执行，停止后不再执行）

2. 定时器版（第一次延迟执行，停止后会再执行一次）

function throttle(fn, delay) {
  let timer = null;
  return function(...args) {
    if (!timer) {
      timer = setTimeout(() => {
        fn.apply(this, args);
        timer = null;                  // 清除定时器，允许下次触发
      }, delay);
    }
  };
}

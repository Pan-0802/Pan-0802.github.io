stl常见容器及算法

发表于2026-01-11|更新于2026-01-18|C++
|总字数:3k|阅读时长:12分钟

常见容器 存储数据的“仓库”

容器主要分为三类：序列式容器、关联式容器和容器适配器。

序列式容器 (Sequence Containers)

强调元素的线性存储。

容器 特点 适用场景
std::vector 动态数组，后端插入快，支持随机访问。 首选容器。适用于频繁读取、末尾增删。
std::list 双向链表，任意位置增删快，不支持随机访问。 频繁在中间插入/删除。
std::deque 双端队列，头尾增删都很快。 需要在两端进行操作。

用法示例

  1. std::vector —— 动态数组（最常用）
    C 对比： 相当于一个会自动扩容的 int arr[]。
  1. std::string —— 字符串（救星）
    C 对比： 彻底替代 char[] 和 char*。你再也不用担心 strcpy 溢出或 strcmp 麻烦了。
  1. std::map —— 键值对/字典
    C 对比： C 语言没有对应结构。它像是一个下标可以是任何类型的数组。\
  1. std::set —— 集合
    特点： 里面的元素会自动排序，且不允许重复。
    用法： 适合用来去重。

常用算法（Algorithm）：处理数据的“工具”

算法通常配合“迭代器（Iterator）”使用。迭代器你可以简单理解为 C 语言中的指针，它指向容器中的某个位置。

使用前需包含 #include 。

  1. std::sort —— 排序
    C 对比： 替代 qsort。比 qsort 更快，语法更简单。
  1. std::find —— 查找
    功能： 在容器中找某个值。
  1. std::reverse —— 反转
    用法： std::reverse(v.begin(), v.end()); 直接把数组倒过来。

总结：从 C 到 C++ STL 的转变

需求 C 语言做法 C++ STL 做法
定义动态数组 int *ptr = malloc(n * sizeof(int)) std::vector v
字符串拼接 strcat(dest, src) (容易溢出) s1 + s2
数组排序 手写快排或 qsort std::sort(v.begin(), v.end())
查找元素 写个 for 循环遍历 std::find(…)
存储姓名->电话 定义结构体数组，自己写遍历查找 std::map<string, string>

建议

1.先学 vector 和 string：这两个能解决你 80% 的数据存储问题。

2.理解迭代器：看到 v.begin() 和 v.end() 就把它当成 C 语言里的 &arr[0] 和 &arr[n]。

3.不要怕 :: 和 <>：std:: 表示“标准库里的”， 表示这个容器里装的是 int 类型。

算法知识点总结

一、 基础数值与操作

1.基础操作：min, max, swap
这些函数处理最基本的数值比较和交换。

2.排序与去重：sort, unique
这是 C++ 最强大的地方。注意 unique 只能去除相邻的重复元素，所以通常先排序再去重。

3.排列组合：next_permutation
这个算法可以自动生成下一个字典序更大的排列。

4.二分查找系列：binary_search, lower_bound
前提条件：数组必须是升序排列的。

核心总结与建议

1.区间规则：STL 算法几乎永远遵循 [开始, 结束) 的原则。对于数组 a[n]，区间就是 a 到 a + n。

2.指针 vs 迭代器：在操作数组时，a + i 就是迭代器。如果你想得到整数下标，只需用“结果指针 - 首地址 a”即可。

3.头文件：记住 #include 是这些工具的家。

容器知识点总结

一、 常用容器总结

  1. 基础序列容器
    string (字符串)：专门处理文本，比 C 语言的 char[] 安全且方便。

vector (动态数组)：不限制长度，会自动根据需要扩容。

deque (双端队列)：支持在开头和末尾都能快速增删元素的数组。

  1. 适配器容器 (工具类)
    queue (队列)：先进先出 (FIFO)。

stack (堆栈)：后进先出 (LIFO)。

priority_queue (优先队列)：本质是“堆”，每次取出的都是优先级最高（默认最大）的元素。

  1. 关联容器 (基于红黑树实现)
    set / multiset：集合。set 元素唯一，multiset 允许重复。内部自动排序。

map / multimap：键值对 (Key-Value)。通过键来存取值。

  1. 特殊容器
    bitset (位集)：用于处理二进制位，比数组更省空间。

二、 代码用法示例

这些例子展示了 C++ 是如何简化 C 语言中复杂数据结构的。

  1. vector：不再需要 malloc 的数组
  1. string：告别 strcat 和 strcmp
  1. map：通过键查找值（类似于查字典）
  1. stack 与 queue：经典的后进先出与先进先出

分类

  1. 序列容器
    string: 专门用于处理字符串的容器。

vector: 动态数组，可以自动根据需要扩容。

deque: 双端队列，支持在头尾两端进行高效的插入和删除。

  1. 容器适配器
    stack / queue: 分别实现“后进先出”和“先进先出”的逻辑。

priority_queue: 优先队列（本质是堆），每次弹出的都是优先级最高（通常是最大）的元素。

  1. 关联容器 (通常基于红黑树实现)
    set / multiset: 集合。set 元素唯一且自动排序；multiset 允许重复。

map / multimap: 通过键值对（Key-Value）存储，按键排序。

  1. 其他
    bitset: 位集，用于高效存储和处理二进制位。

三、 学习建议

容器选择：如果你不确定用什么，默认使用 vector，它是性能最均衡的。

自动排序：如果你需要数据进去后自动排好序且不重复，用 set。

空间优化：如果你在做位运算或开关状态记录，bitset 非常高效。

迭代器 (Iterator)

迭代器是容器与算法之间的“桥梁”。
作用：它是一个用于访问容器内数据的指针，既能像指针一样访问数据，又能保证工程的稳定性。核心方法：begin(): 返回指向容器第一个元素的迭代器。end(): 返回指向容器末尾（最后一个元素之后）的迭代器，遵循左闭右开原则 $[begin, end)$。

常用算法 (Algorithms)

算法通过迭代器对容器内容进行处理。

算法 功能描述 用法示例
min / max 求最小值或最大值 max(a, b)
swap 交换两个数据的值 swap(a, b)
sort 快速排序 sort(a, a + 10)
unique 去重（需先排序，返回去重后的尾地址） unique(a, a + 10)
binary_search 二分查找，判断是否存在 binary_search(a, a + 5, val)
lower_bound 查找第一个大于等于给定值的位置 lower_bound(a, a + 10, 55)
upper_bound 查找第一个大于给定值的位置 upper_bound(a, a + 10, 55)
next_permutation 求下一个全排列 next_permutation(a, a + n)

迭代器与 auto

迭代器：用于访问容器内数据的指针，常用 begin() 指向首元素，end() 指向末尾元素后的位置（遵循左闭右开原则）。

auto (C++11)：自动类型推导。在遍历复杂容器（如嵌套 vector）时，能极大简化代码。

Map 的遍历 (C++11/17)
C++11：通过 item.first 访问键，item.second 访问值。

C++17：支持结构化绑定，直接解构键值。

竞赛相关常识

  1. 时间与空间复杂度
    时间：在竞赛中，计算量在 $10^8$ 数量级以内通常不会超时。常见复杂度有 $O(1), O(\log n), O(n), O(n^2), O(C^n), O(n!)$ 等。
    空间：8位 = 1字节；1024字节 = 1KB；1024KB = 1MB。
    注意：在函数内部定义特大数组可能导致栈溢出（段错误），建议定义为全局变量。
    越界：访问 a[100] 的下标 1000 会导致运行时错误。
  2. 评测机返回结果
    Accepted (AC)：答案正确。
    Wrong Answer (WA)：答案错误。
    Time Limit Exceeded (TLE)：运行超时。
    Memory Limit Exceeded (MLE)：空间超限。
    Runtime Error (RE)：运行时错误（如数组越界、除以0）。
    Compilation Error (CE)：编译错误。
